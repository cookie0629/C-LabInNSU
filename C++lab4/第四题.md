# 任务目标

通过在对象序列化与反序列化任务中的实践，展现对 C++ 语言中模板元编程、动态内存操作、输入/输出以及错误处理等机制的理解。

# 学生技能

- 能够使用模板功能开发 C++ 应用程序
- 能够以 C++ 风格处理错误

# 任务要求

C++ 应用程序项目必须：

- 具备构建系统
- 具备单元测试
- 根据所述技术任务书演示程序行为
- 需要使用面向对象风格，并运用模板类和模板函数来完成此任务

# 准备材料

本任务要求编写一个控制台应用程序，该程序接收一种表示格式的输入数据流，并将其转换为另一种表示格式。

为此需要支持：

- 从输入表示格式读取数据：JSON, TOML, XML
- 将读取的文档转换为抽象语法树
- 将抽象语法树转换为指定的输出表示格式：JSON, TOML, XML

输入表示格式通过应用程序命令行的第一个参数指定。输出表示格式通过应用程序命令行的第二个参数指定。输入文档通过程序的标准输入流 (**stdin**) 提供。输出文档必须输出到程序的标准输出流 (**stdout**)。错误信息需要输出到标准错误流 (**stderr**)。

在此任务中，**要求不使用第三方库**，独立编写 JSON, TOML, XML 表示格式的处理程序。

JSON, TOML, XML 格式中的数据类型转换是明确的——换言之，支持一个最小的、共有的数据类型集合。

表示格式的语法处理必须通过实现 C++ 现代错误处理范式之一，以清晰明了的方式呈现解析错误。
