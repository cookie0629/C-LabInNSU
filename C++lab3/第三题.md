# 预备材料

在此任务中，需要开发一个解释正则表达式并将其应用于字符串的控制台应用程序。

应用程序通过标准输入流 (**stdin**) 接收一个正则表达式字符串。随后，对于收到的每个输入行，需要确定其是否与给定的正则表达式匹配，并在标准输出流 (**stdout**) 中输出 **true** 或 **false**。

### 正则表达式语法

本任务中的正则表达式具有以下语法：

-   字符 "a"..."z", "A"..."Z", "0"..."9", " " (空格) 匹配一个对应的字符
-   字符 "." 匹配任意一个字符
-   字符组使用方括号 "[" 开始，"]" 结束来定义。字符组匹配组内的任意一个字符
    -   "[abc]" - "a" 或 "b" 或 "c"
    -   如果两个字符之间插入符号 "-"，则表示将起始字符到结束字符（包含两端）的所有字符添加到组中（例如："a-d" == "abcd"）
-   修饰符 "*" - 将定义（字符或字符组）的作用扩展到匹配该定义 0 次或多次
    -   ".*" - 任意字符 0 次或多次
    -   "[abc]*" - 任意长度的 "a"、"b"、"c" 字符序列
    -   此修饰符是贪婪的
        -   字符串 "aa" 无法满足表达式 "a*a"，因为整个字符串匹配了 "a*"，没有剩余字符给 "a" 匹配
-   修饰符 "+" - 将定义（字符或字符组）的作用扩展到匹配该定义 1 次或多次
    -   ".+" - 任意字符 1 次或多次
    -   "[abc]+" - 长度大于等于 1 的 "a"、"b"、"c" 字符序列
    -   此修饰符是贪婪的
        -   字符串 "aa" 无法满足表达式 "a+a"，因为整个字符串匹配了 "a+"，没有剩余字符给 "a" 匹配
-   修饰符 "?" - 将定义（字符或字符组）的作用扩展到匹配该定义 0 次或 1 次
    -   ".?" - 任意字符 0 次或 1 次
    -   "[abc]?" - "a" 或 "b" 或 "c" 或 "" (空字符串)
    -   此修饰符是懒惰的
        -   字符串 "a" 可以满足表达式 "a?a"，因为 "a?" 将匹配空字符串 ""，而 "a" 将匹配字符串 "a"
